## ‚ú® Why Dither Forge?

Most online dithering tools give you three sliders and a "Floyd-Steinberg" button. **Dither Forge treats dithering as the craft it is.**

<table>
<tr>
<td width="50%" valign="top">

### ūüé® Authenticity over approximation

Every palette is extracted from real hardware ‚ÄĒ not eyeballed from screenshots. Game Boy DMG, NES, Amiga OCS, MS-DOS CGA modes, NTSC and PAL broadcast standards, CRT phosphor profiles. The colors you'd actually have seen in 1989.

</td>
<td width="50%" valign="top">

### ūüß¨ The full algorithm catalog

Twelve dithering algorithms from the classic Floyd‚ÄďSteinberg to modern additions like Riemersma, Blue Noise and Shiau‚ÄďFan. Each with proper parameter controls ‚ÄĒ not pre-baked presets.

</td>
</tr>
<tr>
<td width="50%" valign="top">

### ūüõ†ÔłŹ Pro workflow built-in

Batch processing, side-by-side comparison, CRT preview with scanlines and analog drift, and an Algorithm Lab to design your own dither from scratch.

</td>
<td width="50%" valign="top">

### ‚ö° Made for the modern web

TypeScript + React + TanStack Start, deployed on Cloudflare Workers. Fast everywhere, no installation needed, runs entirely in the browser.

</td>
</tr>
</table>

## ‚ö° Quick Start

### Prerequisites

- [Bun](https://bun.sh/) `‚Č• 1.0` *(or Node 18+ with npm/pnpm/yarn)*

### Installation

```bash
# Clone the repository
git clone https://github.com/MatteCarro/game-dither-dream.git
cd game-dither-dream

# Install dependencies
bun install

# Start the development server
bun run dev
```

Open **`http://localhost:3000`** and start dithering.

### Available Scripts

```bash
bun run dev        # Start the development server
bun run build      # Build for production
bun run preview    # Preview the production build
bun run lint       # Run ESLint
bun run deploy     # Deploy to Cloudflare Workers
```

> [!TIP]
> Prefer npm or pnpm? Replace `bun` with your package manager ‚ÄĒ all scripts are standard.

## ūüďą Performance

Dither Forge is engineered for real-time interactivity:

- **Canvas-based rendering** ‚ÄĒ direct pixel manipulation, no DOM overhead
- **Web Workers** ‚ÄĒ heavy algorithms run off the main thread *(in progress)*
- **Edge deployment** ‚ÄĒ Cloudflare Workers, latency under 50ms worldwide
- **Zero runtime dependencies** ‚ÄĒ pure browser APIs for the core engine
